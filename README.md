# NeroOps

NeroOps — приватный дневник здоровья и повседневной жизни моего лабрадора Неро. Он работает как
устанавливаемое веб-приложение на iPhone, хранит данные на вашем ноутбуке и
синхронизирует офлайн-записи после восстановления связи.

## Что уже работает

- кормление, прогулки, симптомы, самочувствие, тренировки, лекарства, вес,
  визиты к ветеринару и заметки;
- фотографии и PDF до 12 МБ, максимум пять файлов на запись;
- история, фильтры, редактирование и удаление;
- отчёты по активности, симптомам, оценкам и весу;
- офлайн-очередь на iPhone без дублирования записей;
- ZIP-экспорт и согласованная резервная копия SQLite;
- приватный доступ через Tailscale без открытия портов роутера.

NeroOps фиксирует наблюдения, но не ставит диагнозы и не заменяет ветеринара.

## Локальный запуск

Требуются Python 3.12+, Node.js 24+ и npm.

```bash
make install
make dev
```

Во время разработки:

- PWA: `http://127.0.0.1:5173`
- API и документация: `http://127.0.0.1:8000/docs`

Для запуска собранного приложения одним процессом:

```bash
make run
```

После этого NeroOps доступен на `http://127.0.0.1:8000`.

На macOS собранное приложение также можно запустить двойным кликом по
`start-neroops.command` в корне проекта. Скрипт дождётся запуска сервера и
откроет NeroOps в браузере. Для выключения используйте `stop-neroops.command`;
данные при этом сохраняются.

## Доступ с iPhone

1. Установите Tailscale на Mac и iPhone и войдите в одну личную сеть. Для macOS
   используйте рекомендуемую разработчиками
   [Standalone-версию](https://tailscale.com/docs/concepts/macos-variants).
2. На Mac выполните:

   ```bash
   tailscale serve --bg 8000
   tailscale serve status
   ```

3. Откройте показанный HTTPS-адрес в Safari на iPhone.
4. Нажмите **Поделиться → На экран «Домой» → Добавить**.

FastAPI слушает только `127.0.0.1`; наружу его публикует Tailscale Serve внутри
вашей приватной сети. Ноутбук должен быть включён. Если он спит, новая запись
останется на iPhone и отправится позже.

## Автозапуск на macOS

После проверки ручного запуска установите две пользовательские launchd-службы:

```bash
make install-service
```

Первая запускает NeroOps при входе в macOS и перезапускает после сбоя. Вторая
каждый день в 03:15 создаёт backup. Логи находятся в `logs/`.

Удаление служб:

```bash
make uninstall-service
```

## Данные и резервные копии

Данные находятся в `data/`:

- `neroops.sqlite3` — база;
- `attachments/` — фотографии и PDF.

Создать безопасную копию работающей базы:

```bash
make backup
```

Архив появится в `backups/`. Для восстановления остановите NeroOps, распакуйте
`neroops.sqlite3` и `attachments/` обратно в `data/`, затем запустите приложение.

Пользовательский экспорт из интерфейса отличается от backup: экспорт содержит
удобные JSON/CSV, backup — точную копию базы для аварийного восстановления.

## Docker

```bash
docker-compose up --build
```

Контейнер доступен только на `127.0.0.1:8000`, а `data/` остаётся на хосте.
Если Compose-плагин не установлен, тот же образ можно запустить напрямую:

```bash
docker build -t neroops .
docker run --rm -p 127.0.0.1:8000:8000 -v "$PWD/data:/data" neroops
```

## Структура

```text
backend/neroops/
  api.py                HTTP API и файловые операции
  schemas.py            Pydantic-модели и проверка payload
  models.py             SQLAlchemy-модели
  services/README.md    подробная документация сервисного слоя
  services/reports.py   чистая Python-аналитика
frontend/src/
  App.tsx               экраны и формы PWA
  db.ts                 IndexedDB и офлайн-очередь
  api.ts                синхронизация с FastAPI
```

Лучшие первые Python-задачи для изучения проекта находятся в
`backend/neroops/schemas.py` и `backend/neroops/services/reports.py`: они не
требуют знания SQL и покрываются изолированными тестами.

План самостоятельной Python-разработки находится в [TODO.md](TODO.md), а точные
сигнатуры и правила поведения — в [CONTRACTS.md](CONTRACTS.md).

## Проверки

```bash
make lint
make test
```

`make test` запускает Python-тесты, frontend unit-тесты и браузерный сценарий
офлайн-синхронизации через установленный Google Chrome.

Миграции схемы находятся в `migrations/`. Для будущих изменений:

```bash
.venv/bin/alembic revision --autogenerate -m "describe change"
.venv/bin/alembic upgrade head
```
